Explanation

flex-shrink is 1 by default, so in order for one item to "take all the shrinking" we need to first reset it to 0 for all other items.

Once we have one item which "takes all the shrinking" we need to be in a situation where shrinking can occur:

- The flex container's width is less then items' width
- The items are not wrapping (flex-wrap: no-wrap; - default)
- The items are not overflowing the container (either container has overflow: hidden; or another constraint like max-width: 100% and parent less than items' width)
- The item itself can lose pixels (either computed flex-base is more than content width or item has overflow: hidden;)
